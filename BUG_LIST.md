# 量化交易系统 - 完整Bug清单

## 🔴 严重问题（需立即修复）

### 1. 日志文件过大 - trade_signals.json (226MB+) ⚠️ CRITICAL
- **位置**: `data/trade_signals.json`
- **问题**: 84万+条日志记录，文件已达226MB，持续增长中
- **影响**: 
  - 磁盘空间被快速占满
  - 日志读取速度极慢
  - 可能导致系统崩溃
- **修复建议**: 
  - 添加日志轮转机制（按天或按大小分割）
  - 定期清理旧日志（保留最近30天）
  - 压缩历史日志

## 🟡 中等问题（建议尽快修复）

### 2. 数据库连接未使用连接池
- **位置**: `trading_core/order_executor.py`
- **问题**: 每次数据库操作都新建连接，高并发时性能差
- **代码位置**: `_record_trade()`, `_record_signal()`, `get_recent_trades()` 等方法
- **修复建议**: 使用连接池或单例连接

### 3. 内存持仓与交易所持仓不同步
- **位置**: `trading_core/order_executor.py` 的 `self.positions`
- **问题**: 内存中的持仓跟踪可能与交易所实际持仓不一致
- **影响**: 止盈止损监控可能监控错误的持仓
- **修复建议**: 定期与交易所同步持仓状态

### 4. Signal Memory 文件路径问题
- **位置**: `strategy/strategy_engine.py` 的 `MEMORY_FILE = "signal_memory.json"`
- **问题**: 使用相对路径，可能因工作目录不同而找不到文件
- **修复建议**: 使用绝对路径，与 `data` 目录统一管理

### 5. 配置文件不一致
- **位置**: `config.py` vs `web_admin/app.py`
- **问题**: 
  - `config.py` 使用 `API_KEY` / `API_SECRET`
  - `web_admin/app.py` 使用 `BINANCE_API_KEY` / `BINANCE_SECRET_KEY`
- **影响**: 配置混乱，可能导致API连接失败
- **修复建议**: 统一配置项名称

### 6. 策略引擎异常处理过于简单
- **位置**: `strategy/strategy_engine.py` 的 `check_ma99_strategy()`
- **问题**: 异常时直接返回 `None, None, None`，丢失错误信息
- **修复建议**: 添加详细错误日志
