我建议你把这次升级拆成 3 层：
第一层：只加后台展示和配置，不动核心下单逻辑
先做：

二、交易记录页面改造方案
你的要求是：
展示已完成的所有交易 
部分平仓也要展示 
这个点很重要，你现在不能再只把“订单”当成记录，而要把“成交/执行事件”单独记录。

2.1 新增交易执行明细表
你现在如果只有 positions / orders 这种表，不够。
必须新增一张：
trade_fills
每一笔实际执行都记录，不管是：
开仓 
加仓 
部分平仓 
全部平仓 
止盈 
止损 
手动平仓 
建议字段：
CREATE TABLE IF NOT EXISTS trade_fills (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    strategy_name TEXT,
    symbol TEXT NOT NULL,
    side TEXT NOT NULL,
    position_side TEXT,
    action_type TEXT NOT NULL,   -- open / add / partial_close / close / take_profit / stop_loss / manual_close
    order_id TEXT,
    exchange_trade_id TEXT,
    quantity REAL NOT NULL,
    price REAL NOT NULL,
    realized_pnl REAL DEFAULT 0,
    fee REAL DEFAULT 0,
    fee_asset TEXT,
    ai_model TEXT,
    ai_decision TEXT,
    signal_source TEXT,
    signal_reason TEXT,
    executed_at TEXT NOT NULL,
    created_at TEXT NOT NULL
);

2.2 为什么必须加 trade_fills
因为你说“部分平仓的数据也要展示”。
如果你只看 position 表，部分平仓只会改变仓位数量，历史细节会丢。
所以要把每一次执行动作都记下来。

开仓成功后 
部分平仓成功后 
止盈触发后 
止损触发后 
手动平仓后 
建议新增一个私有方法：
def _record_trade_fill(self, fill_data: dict):
    ...
然后所有执行动作统一调用这个方法。

六、我给你的更合适建议
你原来的想法整体是对的，但我给你 4 个更稳的补充建议。

建议 1：仪表盘不要一开始做成全市场联动
更好做法：
先支持 10~20 个主流币种选择 
当前页只渲染当前币种 
后续再扩展自选列表 
这样性能和稳定性都更可控。

建议 2：自动交易开关要分成 2 个
不要只有一个“总开关”。
建议分成：
策略扫描开关 
自动下单开关 
这样即使你关了自动下单，系统仍然可以继续盯盘和发 Telegram。

建议 3：AI 模型要按策略绑定，不要做全局唯一模型
这是最灵活的方式。
例如：
BTC 趋势策略 → GPT 
ETH 短线策略 → DeepSeek 
SOL 新闻策略 → Claude 

建议 4：交易记录页要把“订单”和“成交”分开
因为：
订单是意图 
成交是事实 
你要看的“部分平仓”，本质是成交事件，不是订单摘要。

九、最后给你的结论
你的这 4 个需求都合理，而且都能做。
但为了“不造成量化系统崩溃无法使用”，最重要的是不要一步到位大重构。
