# 策略引擎启动问题解决方案

## 问题
策略已经21小时没有运行，最后扫描时间显示为21小时前。

## 原因
Web服务器（Flask）和策略引擎是两个独立的进程：
1. Web服务器负责提供Web界面和API
2. 策略引擎负责扫描市场和执行交易

当前的Web服务器只初始化了策略管理器，但没有启动策略扫描循环。

## 解决方案

### 方案1：独立启动策略引擎（推荐）

在服务器上运行以下命令启动策略引擎：

```powershell
cd C:\Users\TUF\.openclaw\workspace\trading_system
.\venv\Scripts\python run_strategy.py
```

这个脚本会：
1. 初始化交易所连接
2. 初始化风险管理器
3. 初始化订单执行器
4. 初始化策略管理器
5. 启动策略扫描循环（每60秒扫描一次）

### 方案2：通过Web界面启动

在信号监控页面点击"▶️ 启动策略"按钮。

注意：这个方案需要Web服务器的API能够正确启动策略引擎。如果失败，请使用方案1。

### 方案3：重启Web服务器（自动启动）

修改后的 `app.py` 已经添加了自动启动功能。重启Web服务器后，策略引擎会自动启动。

```powershell
# 停止当前的Web服务器（找到Python进程并结束）
# 然后重新启动
.\venv\Scripts\python -m web_admin.app
```

## 修改的文件

1. `web_admin/app.py` - 添加策略引擎启动/停止API和自动启动功能
2. `web_admin/templates/signals.html` - 添加启动/停止按钮和状态警告
3. `run_strategy.py` - 新增独立启动策略引擎脚本

## 验证策略是否运行

启动后，可以通过以下方式验证：

1. 查看信号监控页面的"策略运行状态"面板
2. 观察"最后扫描时间"是否在更新
3. 查看"策略监控信号日志"是否有新的SCAN日志
4. 检查"扫描次数"是否在增加

## 注意事项

- 策略引擎需要持续运行才能扫描市场和生成信号
- 如果关闭运行策略引擎的窗口，策略会停止
- 建议将策略引擎作为后台服务运行（可以使用pm2、systemd等工具）
